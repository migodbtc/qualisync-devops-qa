## 🏗️ Project Overview

**Qualisync** serves as both a *study* and a *personal application* of integrating multiple IT disciplines. It bridges development-focused areas—such as **fullstack development**, **REST API design**, and **UI/UX engineering**—with operations-oriented practices like **version control**, **quality assurance** (both automated and manual), and **containerization/cloud environments**.

With a background rooted in *software development*, I have spent the past **four years** exploring modern development practices, engaging with industry professionals, and staying current with technological trends. While software development has become more accessible, its relative value has shifted, highlighting the growing importance of **operational expertise** in today’s IT landscape.

This project reflects the *evolving demands* of the industry: developers are now expected to understand not only how to build software, but also how to **manage infrastructure**, **automate processes**, **ensure quality**, and support **continuous integration and delivery (CI/CD)**. Although this project may not encompass every technology or aspect of large-scale industry experience, it is designed to expand my skills beyond development—*deepening my understanding* of software engineering as a whole and preparing me for the challenges of modern IT.

## 🏢 Fuchsia – Qualisync’s Fictional ATMS Web Application

To fully leverage **version control**, **quality assurance**, **Docker containerization**, and other modern IT practices, this project includes a *(continuously evolving)* web application that serves as both the main client-facing product and a foundation for operational experimentation. **Fuchsia** addresses this need—a web application designed for a *fictional apartment-tenant management system*, inspired by my previous academic and personal projects.

**Fuchsia** acts as a centralized digital hub for *tenants*, *administrators* (apartment owners and staff), and *room management*. It streamlines operations with features such as **role-based access control (RBAC)**, **intuitive UI/UX**, **room and tenant management**, **payment and lease tracking**, and more. The architecture is built on **Next.js** for the frontend, a **Flask API** backend, and **MySQL** for data storage. Supporting technologies include **Vitest** and **Playwright** for testing, **Tailwind CSS** for styling, and **Docker** for containerization. This setup provides a practical environment to explore and apply both development and operations concepts in a real-world-inspired scenario.

## ⏳ Project Duration
This project was initially made to be a *one-off project*, to be rapidly developed under one month but ended up being almost 6 weeks of development due to my own ignorances and naivety towards the complexity of **quality assurance engineering**, **YAML automation**, **Docker images & terminologies**, and understanding **configuration management**, **virtualization**, **orchestration**, and so much more. I could not also find the time to be able to consistently work on this anymore as there are already two ongoing projects that I have to attend to within my own internship. The first 3 weeks of the internship were a preparation phase for all departments, and now that the dust has cleared and the plan has been constructed for all departments, I wouldn't have the same time as I had during the first 3 weeks. However, I am happy with what the project has achieved so far and may revisit it in the future if time permits.
